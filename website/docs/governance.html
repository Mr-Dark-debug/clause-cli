AI Governance

Define clear rules that guide AI behavior in your projects ‚Äî ensuring consistency, quality, and architecture compliance.

What is AI Governance?

AI Governance is Clause's core differentiator. It's a system of markdown files and structured data that live inside your project, teaching AI coding assistants how to work in your codebase.

Without governance, AI assistants make ad-hoc decisions about naming conventions, file organization, architecture patterns, and error handling. With Clause's governance, every AI interaction follows your team's established rules.

Why It Matters

  • Consistency ‚Äî Every AI-generated file follows the same patterns
  • Quality ‚Äî Coding standards are enforced automatically
  • Architecture ‚Äî AI respects your architectural decisions
  • Onboarding ‚Äî New team members (and their AI tools) instantly understand the project

System Prompts

The system_prompt.md file is the most important governance file. It acts as a persistent instruction set for AI assistants. Here's what it typically contains:

  • Project overview and purpose
  • Coding standards (naming, formatting, commenting)
  • File organization rules
  • Forbidden patterns and anti-patterns
  • Testing requirements
  • Error handling conventions
markdown
# System Prompt ‚Äî My Project

## Coding Standards
- Use TypeScript strict mode
- All functions must have JSDoc comments
- Use named exports, never default exports
- Error handling: always use Result types

## Architecture
- Follow hexagonal architecture
- Keep domain logic pure (no I/O)
- All side effects through ports/adapters

Brainstorm System

The brainstorm.md file is Clause's unique self-reflection mechanism. Before writing code, AI agents can write their reasoning into this file ‚Äî what they're planning, what constraints they see, and what alternatives they considered.

How It Works

  1. Read ‚Äî AI reads existing brainstorm notes for context
  2. Think ‚Äî AI writes its plan and reasoning
  3. Act ‚Äî AI writes code based on the brainstorm
  4. Record ‚Äî Decisions are preserved for future reference

Best Practices

  • Encourage AI to brainstorm before complex changes
  • Review brainstorm entries during code review
  • Archive old entries periodically to keep the file focused

Component Registry

The component_registry.json file tracks all known components in your project. AI assistants use this to avoid creating duplicate components and to understand relationships between existing ones.

json
{
  "components": [
    {
      "name": "Button",
      "path": "src/components/ui/Button.tsx",
      "type": "ui",
      "description": "Primary button component with variants"
    },
    {
      "name": "UserService",
      "path": "src/services/UserService.ts",
      "type": "service",
      "description": "Handles user CRUD operations"
    }
  ]
}

Custom Governance Rules

You can extend governance by creating additional markdown files in the ai_prompt_guidelines/ directory. Clause will include them in the AI's context.

bash
# Add a custom governance rule
clause add governance --name "testing-rules"
# Creates ai_prompt_guidelines/testing-rules.md

Example custom rule:

markdown
# Testing Rules

## Unit Tests
- Every function must have at least one test
- Use describe/it pattern
- Mock external dependencies

## Integration Tests
- Test API endpoints with supertest
- Use test database, never production
