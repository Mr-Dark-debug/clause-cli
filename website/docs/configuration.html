Configuration

Customize Clause's behavior through config files and environment variables.

Config File

Clause looks for a configuration file in these locations (in order):

  1. .clause.yaml in the current directory (project-level)
  2. ~/.config/clause/config.yaml (user-level)

Example Configuration

yaml
# .clause.yaml
default_template: go-rest-api
governance:
  enabled: true
  strict: false
  custom_rules_dir: ./rules

git:
  auto_init: true
  initial_branch: main

output:
  colored: true
  verbose: false

Environment Variables

All config options can be overridden with environment variables prefixed with CLAUSE_:

bash
export CLAUSE_DEFAULT_TEMPLATE=react-app
export CLAUSE_GOVERNANCE_STRICT=true
export CLAUSE_GIT_AUTO_INIT=false

All Options

Option Default Description
default_template auto Default template for new projects
governance.enabled true Enable AI governance generation
governance.strict false Treat validation warnings as errors
git.auto_init true Auto-initialize git repository
output.colored true Enable colored CLI output
