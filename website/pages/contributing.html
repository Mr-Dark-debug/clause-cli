Contributing to Clause

Thank you for your interest in contributing! Every contribution, from bug reports to code, makes Clause better.

Code of Conduct

We are committed to providing a welcoming and inclusive experience for everyone. Please read our Code of Conduct before participating.

Development Setup

Prerequisites

  • Go 1.21 or later
  • Git
  • Make

Building from Source

bash
# Clone the repository
git clone https://github.com/Mr-Dark-debug/clause-cli.git
cd clause-cli

# Install dependencies
go mod download

# Build
make build

# Run tests
make test

# Run linter
make lint

Coding Standards

  • Follow the Effective Go guidelines
  • Run gofmt and golangci-lint before committing
  • Write tests for all new functionality
  • Document exported functions and types

Commit Guidelines

We use Conventional Commits:

text
feat: add new template for Rust CLI apps
fix: correct path handling on Windows
docs: update installation guide for Snap
refactor: simplify governance validation logic
test: add integration tests for init command
chore: update Go dependencies

Release Process

Releases are automated via GoReleaser. When a new tag is pushed, the CI/CD pipeline builds and publishes binaries for all platforms.

  1. Version bump in go.mod and changelog
  2. Create a git tag: git tag v1.x.x
  3. Push the tag: git push origin v1.x.x
  4. GoReleaser handles the rest
